# 🎙️ Voice Assistant — голосовой RAG-ассистент

Бесплатный голосовой ассистент: говоришь вопрос — получаешь **голосовой ответ**
по своей базе знаний, с цитированием источника. Работает полностью на
бесплатных компонентах: распознавание и озвучка — локально на MacBook,
генерация ответа — через бесплатные облачные модели провайдера Ollama.

> Ни одного платного API-ключа. STT (Whisper) и TTS (macOS `say`) работают
> офлайн, генерация — через free-tier модели.

---

## Что это умеет

- 🎤 **Голосовой ввод** — записываешь вопрос в браузере или терминале.
- 🧠 **RAG по твоей базе** — ассистент ищет ответ в твоих документах (Markdown/текст), а не выдумывает.
- 🔊 **Голосовой ответ** — озвучивается через штатный macOS `say` (динамики ноутбука).
- 💬 **Текстовый режим** — можно просто напечатать вопрос.
- 📚 **С цитатами** — каждый ответ ссылается на файл-источник (без галлюцинаций).

---

## Требования

| Что | Зачем | Проверка |
|-----|-------|----------|
| **macOS** (Intel или Apple Silicon) | `say` встроен только в macOS | `say "привет"` в терминале |
| **Node.js 18+** | рантайм TypeScript | `node -v` |
| **Homebrew** | установка sox/ffmpeg/whisper | `brew -v` |
| **Ollama** (локально запущена) | embeddings + fallback-модели | `ollama list` |
| **whisper.cpp** | локальное распознавание речи | `which whisper-cli` |
| **Браузер** Chrome/Safari | микрофон для `localhost` | — |

> 💡 **Про браузер:** Яндекс.Браузер и некоторые другие могут молча блокировать
> микрофон для `localhost`. Если запрос на доступ не появляется — используй
> **Chrome** или **Safari**, либо вручную разреши микрофон для `localhost:3000`
> в настройках сайта браузера.

## Установка с нуля (пошагово)

### Шаг 1. Клонирование и зависимости Node

```bash
git clone https://github.com/bestdeejay-design/voice-assistant.git
cd voice-assistant
npm install
```

### Шаг 2. Системные утилиты (через Homebrew)

```bash
brew install sox          # запись аудио с микрофона (консольный режим)
brew install ffmpeg       # конвертация браузерного аудио в wav для Whisper (веб-режим)
brew install whisper-cpp  # whisper-cli — локальное распознавание речи
```

После установки проверь:

```bash
which whisper-cli      # должно вывести путь, напр. /opt/homebrew/bin/whisper-cli
```

> Если `whisper-cli` установлен в нестандартное место — укажи его через
> переменную `WHISPER_BIN` (см. «Конфигурация»).

### Шаг 3. Модель Whisper

Скачай модель распознавания (≈480 МБ, один раз):

```bash
mkdir -p models
curl -L -o models/ggml-small.bin \
  https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-small.bin
```

Проверь:

```bash
ls -lh models/ggml-small.bin   # должен быть ~480 МБ
```

> Хочешь быстрее/точнее? Возьми `ggml-medium.bin` (~1.5 ГБ) или `ggml-base.bin`
> (~140 МБ) с того же HuggingFace-репозитория и укажи путь через `WHISPER_MODEL`.

### Шаг 4. Ollama и модели

Установи Ollama (https://ollama.com), запусти его, затем скачай модели:

```bash
# локальные модели
ollama pull nomic-embed-text     # embeddings (векторизация твоих доков)
ollama pull deepseek-r1          # fallback генерации, если облако недоступно

# облачные free-tier модели провайдера opencode (нужен запущенный opencode)
ollama pull minimax-m3:cloud
ollama pull nemotron-3-super:cloud
```

Проверь, что Ollama отвечает:

```bash
curl http://localhost:11434/api/tags   # должен вернуть JSON со списком моделей
```

> ⚠️ **Облачные модели `:cloud`** берутся от провайдера Ollama (opencode).
> Если у тебя только локальный Ollama — они не появятся. В этом случае
> ассистент автоматически переключится на локальный `deepseek-r1`
> (см. fallback-цепочку). Можно задать свою модель через `GEN_MODEL`.

### Шаг 5. Твоя база знаний

Положи свои документы (`.md`, `.txt`) в любую папку. Например:

```bash
mkdir -p ~/my-knowledge/docs
# скопируй туда свои .md / .txt файлы
```

### Шаг 6. Индексация базы

```bash
KB_SOURCE=~/my-knowledge/docs npm run ingest
```

Команда прочитает все документы, векторизует их через `nomic-embed-text`
и сохранит индекс в `.kb-index.json`. Запускай её каждый раз, когда
меняешь документы.

Проверка:

```bash
ls -lh .kb-index.json     # файл появился
```

## Запуск

### Веб-интерфейс (рекомендуется)

```bash
npm run web
```

Открой **http://localhost:3000** в Chrome или Safari.

**Как пользоваться:**
- **🎤 Тест микрофона** — проверить доступ к микрофону (показывает ошибку, если заблокирован).
- **🎙️ Записать** — говоришь → **■ Стоп** → видишь расшифровку вопроса и ответ с цитатами.
- Ответ **автоматически озвучивается** через macOS `say` (динамики ноутбука).
- **🔊 Озвучить** под ответом — проиграть ещё раз.
- **Поле внизу** — написать вопрос текстом и Enter (работает без микрофона).

Аудио из браузера передаётся на сервер в памяти (без записи файлов на диск),
конвертируется `ffmpeg` → wav и подаётся в Whisper через stdin.

> 🔒 **Доступ к микрофону:** при первом запуске macOS спросит разрешение
> для браузера (Системные настройки → Конфиденциальность → Микрофон).
> Если запрос не появляется и статус красный «Permission denied» — зайди
> в настройки сайта `localhost:3000` в браузере и вручную разреши микрофон.

### Консольный режим

```bash
npm run assist
```

Интерактивное управление микрофоном в терминале:
- **`r`** — начать запись
- **`s`** — остановить и получить ответ (расшифровка + озвучка)
- **`q`** — выход

Обработка готового аудиофайла (wav, 16 kHz, mono):

```bash
npm run assist file путь/к/файлу.wav
```

```

│   └── server.ts       — веб-сервер + чат-UI (npm run web)

│   └── index.html      — интерфейс: запись, текстовый ввод, озвучка
├── scripts/
│   └── ingest.ts       — индексация базы знаний
├── models/
│   └── ggml-small.bin  — Whisper-модель (скачивается на Шаге 3)
├── docs/
│   └── PROJECT.md      — описание проекта
├── .kb-index.json      — векторный индекс (генерится при ingest, в .gitignore)
├── .last-answer.txt    — последний ответ (для просмотра, в .gitignore)
└── README.md
```

## Конфигурация (переменные окружения)

Все настройки — через переменные окружения. Пример:

```bash
KB_SOURCE=~/my-knowledge/docs \
WHISPER_LANG=en \
TTS_VOICE=Alex \
GEN_MODEL=deepseek-r1 \
npm run web
```

| Переменная | Назначение | Дефолт |
|------------|-----------|--------|
| `KB_SOURCE` | папка с исходниками знаний | `/Users/best/Projects/lovii_docs/docs` |
| `GEN_MODEL` | модель генерации (переопределяет fallback) | `minimax-m3:cloud` |
| `OLLAMA_HOST` | адрес Ollama | `http://localhost:11434` |
| `WHISPER_BIN` | путь к `whisper-cli` | `/opt/homebrew/bin/whisper-cli` |
| `WHISPER_MODEL` | путь к модели whisper | `./models/ggml-small.bin` |
| `WHISPER_LANG` | язык распознавания | `ru` |
| `TTS_VOICE` | голос macOS `say` | `Yuri` |
| `INDEX_PATH` | файл векторного индекса | `./.kb-index.json` |
| `PORT` | порт веб-сервера | `3000` |

Список голосов `say`:

```bash
say -v '?'      # выведет все доступные голоса (выбери свой для TTS_VOICE)
```

**Fallback-цепочка генерации:**
`minimax-m3:cloud` → `nemotron-3-super:cloud` → `deepseek-r1` → `yi-coder`.
При недоступности облака ассистент автоматически переключается на локальную модель.

## Как это работает (архитектура)

```
┌─────────────┐   ┌──────────────┐   ┌────────────────────┐   ┌──────────────┐
│  Микрофон   │ → │  Whisper STT │ → │   RAG (retrieval)   │ → │  LLM (генерация)│
│  (MacBook)  │   │ whisper.cpp  │   │ Ollama embeddings + │   │ облачная модель│
└─────────────┘   └──────────────┘   │ локальный индекс    │   └──────┬───────┘
                                       └────────────────────┘          │
                                                 ↑ цитаты              ↓ ответ
                                       ┌────────────────────┐   ┌──────────────┐
                                       │  macOS «say» (TTS) │ ← │  динамики    │
                                       └────────────────────┘   └──────────────┘
```

| Этап | Компонент | Где работает |
|------|-----------|--------------|
| STT (речь → текст) | whisper.cpp (`whisper-cli`) + `ggml-small.bin` | локально, офлайн |
| Embeddings | Ollama `nomic-embed-text` | локально |
| Векторный индекс | in-process (косинусное сходство, файл `.kb-index.json`) | локально, zero-infra |
| LLM (генерация) | `minimax-m3:cloud` / `nemotron-3-super:cloud` (fallback: `deepseek-r1`, `yi-coder`) | облачно (free-tier) |
| TTS (текст → речь) | macOS `say` | локально, встроено |

## Troubleshooting

| Симптом | Причина / решение |
|---------|-------------------|
| `say` не озвучливает | macOS глушит звук? Проверь `say "тест"`. Голос задаётся `TTS_VOICE`. |
| `whisper-cli: command not found` | Не установлен whisper-cpp (Шаг 2) или не в PATH. Задай `WHISPER_BIN`. |
| «не удалось распознать речь» | Тихо / шумно / модель не скачана. Проверь `models/ggml-small.bin`. |
| Браузер не запрашивает микрофон | Используй Chrome/Safari или разреши микрофон для `localhost:3000` вручную. |
| `Permission denied` на микрофон | В настройках сайта `localhost:3000` стоит «Запретить» — переключи в «Разрешить». |
| `curl http://localhost:11434` не отвечает | Ollama не запущена. Запусти приложение Ollama. |
| Генерация падает / пустой ответ | Нет ни облачных, ни локальной модели. `ollama pull deepseek-r1`. |
| Ответы не по теме / галлюцинации | Переиндексируй базу: `KB_SOURCE=... npm run ingest`. |

## Особенности

- **Бесплатно**: нет платных API-ключей, STT/TTS локально, генерация — free-tier.
- **Офлайн-часть**: распознавание и озвучка работают без интернета.
- **С цитатами**: ответ всегда ссылается на источник из базы (без галлюцинаций).
- **Heuristic boost**: при упоминании «архитектура/дизайн/финансы» подтягивается
  соответствующий документ из индекса для точности.
- **Расширяемо**: легко заменить TTS на Piper/Coqui, добавить чат-бот интерфейс
  или переключить генерацию на любую другую модель Ollama.
