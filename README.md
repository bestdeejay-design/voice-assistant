# 🎙️ Voice Assistant — голосовой RAG-ассистент

Бесплатный голосовой ассистент: говоришь вопрос — получаешь голосовой ответ
по базе знаний проекта **Lovii**, с цитированием источника. Работает
полностью на бесплатных компонентах: распознавание и озвучка — локально,
генерация ответа — через бесплатные облачные модели.

> Не нужен ни один платный API-ключ. STT и TTS работают офлайн на MacBook,
> генерация — через free-tier модели Ollama-провайдера.

---

## Демо-сценарий

```
Пользователь: «Расскажи про проект Лови»
  ↓ (микрофон)
Whisper распознаёт речь → «расскажи про проект лови»
  ↓
RAG ищет похожие фрагменты в базе знаний (Ollama embeddings + локальный индекс)
  ↓
Облачная модель генерирует краткий ответ по контексту + цитаты
  ↓
macOS «say» озвучивает ответ через динамики
```

Пример ответа (текст, который слышит пользователь):

> Проект **LOVII** — платформа лояльности для локальных сообществ. Баллы
> начисляются по курсу 1 балл = 1 ₽, не сгорают. Сейчас идёт подготовка к
> «первому рублю»: нет оплаты в приложении и UI лояльности — это ключевые
> разрывы. *Источник: PARAMS.md, START_VECTOR.md*

## Архитектура

```
┌─────────────┐   ┌──────────────┐   ┌────────────────────┐   ┌──────────────┐
│  Микрофон   │ → │  Whisper STT │ → │   RAG (retrieval)   │ → │  LLM (генерация)│
│  (MacBook)  │   │ whisper.cpp  │   │ Ollama embeddings + │   │ облачная модель│
└─────────────┘   └──────────────┘   │ локальный индекс    │   └──────┬───────┘
                                      └────────────────────┘          │
                                                ↑ цитаты              ↓ ответ
                                      ┌────────────────────┐   ┌──────────────┐
                                      │  macOS «say» (TTS) │ ← │  динамики    │
                                      └────────────────────┘   └──────────────┘
```

| Этап | Компонент | Где работает |
|------|-----------|--------------|
| STT (речь → текст) | whisper.cpp (`whisper-cli`) + `ggml-small.bin` | локально, офлайн |
| Embeddings | Ollama `nomic-embed-text` | локально |
| Векторный индекс | in-process (косинусное сходство, файл `.kb-index.json`) | локально, zero-infra |
| LLM (генерация) | `minimax-m3:cloud` / `nemotron-3-super:cloud` (fallback: `deepseek-r1`, `yi-coder`) | облачно (free-tier) |
| TTS (текст → речь) | macOS `say` (голос Yuri) | локально, встроено |

## Стек (100% бесплатно)

- **TypeScript + Node.js** — рантайм
- **whisper.cpp** — локальное распознавание речи (модель `ggml-small.bin`, ~480 МБ)
- **Ollama** — embeddings (`nomic-embed-text`) + fallback-модели генерации
- **Облачные free-tier модели** Ollama-провайдера — генерация ответа
- **macOS `say`** — озвучка, без установки и моделей

## Быстрый старт

### 1. Зависимости системы

```bash
brew install sox          # запись аудио с микрофона (консольный режим)
brew install ffmpeg       # конвертация браузерного аудио в wav для Whisper (веб-режим)
```

Ollama должна быть запущена и содержать модели:
`nomic-embed-text`, `deepseek-r1` (fallback), а также доступ к облачным
`minimax-m3:cloud` / `nemotron-3-super:cloud` через провайдер opencode.

Whisper-модель кладётся в `models/ggml-small.bin` (уже в репозитории не входит,
скачивается отдельно — см. ниже).

### 2. Установка

```bash
npm install
```

### 3. Индексация базы знаний

По умолчанию индексируется `/Users/best/Projects/lovii_docs/docs`.
Путь можно переопределить через `KB_SOURCE`:

```bash
npm run ingest                       # дефолтный источник
KB_SOURCE=/путь/к/твоим/докам npm run ingest
```

### 4. Запуск голосового ассистента

#### Веб-интерфейс (рекомендуемый)

```bash
npm run web
```

Открой **http://localhost:3000**. В браузере:

- **🎙️ Записать** — говоришь в микрофон, жмёшь **■ Стоп**;
- вопрос расшифровывается и показывается в чате, рядом — ответ с цитатами;
- ответ **автоматически озвучивается на ноутбуке** через macOS `say`;
- **🔊 Озвучить** под каждым ответом — проиграть ещё раз;
- **поле ввода** внизу — можно написать вопрос текстом и нажать **➤**.

Аудио из браузера передаётся на сервер в памяти (без записи файлов на диск),
конвертируется ffmpeg → wav и подаётся в Whisper через stdin.

> Для микрофона в браузере macOS запросит доступ для браузера
> (Системные настройки → Конфиденциальность → Микрофон).

#### Консольный режим

```bash
npm run assist
```

Интерактивное управление микрофоном:
- **`r`** — начать запись, **`s`** — остановить и получить ответ, **`q`** — выход.

Обработка готового аудиофайла (wav, 16 kHz, mono):

```bash
npm run assist file путь/к/файлу.wav
```

## Конфигурация (переменные окружения)

| Переменная | Назначение | Дефолт |
|------------|-----------|--------|
| `KB_SOURCE` | папка с исходниками знаний | `/Users/best/Projects/lovii_docs/docs` |
| `GEN_MODEL` | модель генерации (переопределяет fallback) | `minimax-m3:cloud` |
| `OLLAMA_HOST` | адрес Ollama | `http://localhost:11434` |
| `WHISPER_BIN` | путь к `whisper-cli` | `/opt/homebrew/bin/whisper-cli` |
| `WHISPER_MODEL` | путь к модели whisper | `./models/ggml-small.bin` |
| `WHISPER_LANG` | язык распознавания | `ru` |
| `TTS_VOICE` | голос macOS `say` | `Yuri` |
| `INDEX_PATH` | файл векторного индекса | `./.kb-index.json` |

**Fallback-цепочка генерации:**
`minimax-m3:cloud` → `nemotron-3-super:cloud` → `deepseek-r1` → `yi-coder`.
При недоступности облака ассистент автоматически переключается на локальную модель.

## Особенности

- **Бесплатно**: нет платных API-ключей, STT/TTS локально, генерация — free-tier.
- **Офлайн-часть**: распознавание и озвучка работают без интернета.
- **С цитатами**: ответ всегда ссылается на источник из базы (без галлюцинаций).
- **Heuristic boost**: при упоминании «архитектура/дизайн/финансы» подтягивается
  соответствующий документ из индекса для точности.
- **Расширяемо**: легко заменить TTS на Piper/Coqui, добавить чат-бот интерфейс
  или переключить генерацию на любую другую модель.
