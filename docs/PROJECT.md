# voice-assistant

Голосовой RAG-ассистент: распознаёт речь (Whisper), ищет ответ в базе знаний
проекта Lovii (локальный RAG) и озвучивает ответ (macOS TTS). Полностью
бесплатно и локально — без платных API-ключей и облачных сервисов.

## Назначение

Пользователь задаёт вопрос голосом по документации проекта (архитектура,
дизайн, PRD, финансовая модель, юридическая структура). Ассистент возвращает
краткий ответ с указанием источника — без галлюцинаций, только по базе.

## Стек (100% бесплатно / локально)

| Слой | Решение | Почему бесплатно |
|------|---------|------------------|
| STT (распознавание) | whisper.cpp `whisper-cli` + `ggml-small.bin` | локально, офлайн |
| Embeddings | Ollama `nomic-embed-text` | локально, без ключа |
| Векторный индекс | локальный in-process (косинусное сходство, файл) | zero-infra |
| LLM (генерация) | Ollama `deepseek-r1` | локально, без ключа |
| TTS (озвучка) | macOS `say` (голос Yuri) | встроен в macOS |
| Рантайм | TypeScript + Node.js v25 | бесплатно |

## Архитектура

```
[Микрофон] → stt.ts (Whisper) → текст вопроса
   → rag.ts: querySimilar() (Ollama embeddings + локальный индекс)
             → генерация (Ollama deepseek-r1) + цитаты
   → tts.ts (say) → [Динамики]
```

- **Ingestion**: `scripts/ingest.ts` читает `lovii_docs/docs`, режет на чанки,
  эмбеддит в локальный индекс. Исключает служебные папки `.git/`, `.omo/`.
- **STT**: whisper-cli пишет `.txt` рядом с входным аудио, читаем результат.
- **Cost = 0**: всё локально, платных ключей нет.

## Дизайн-решения

- STT через whisper.cpp (brew) вместо облачных API — офлайн, бесплатно.
- TTS через встроенный macOS `say` — ноль установки и моделей, достаточно для демо.
- Локальный in-process индекс вместо серверной векторной БД — zero-infra.
- RAG переиспользует тот же подход, что и в сервисе knowledge-base.

## Результат

- Рабочий голосовой ассистент: вопрос голосом → ответ голосом по базе знаний.
- Ответ с цитированием источника, без галлюцинаций.
- Работает полностью офлайн на одной машине, бесплатно.
